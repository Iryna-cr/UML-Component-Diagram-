# UML-Component-Diagram-

## Опис:
Діаграма компонентів відображає архітектуру системи бронювання квитків у кіно, поділену на три основні шари: Presentation, Business Logic та Data Access. Кожен шар має свої компоненти та чітко визначену роль у системі.

## Основні шари системи:
### 1)Presentation

Цей шар відповідає за взаємодію користувача із системою.
Сюди входять:

BookingForm — форма бронювання (користувач обирає фільм, час, місця).

LoginForm — форма авторизації (вхід до системи).

Основна функція: отримання введення користувача і відображення результатів.
Залежить від шару Business Logic.

### 2)Business Logic

Цей шар реалізує основні правила і логіку системи.
Він містить сервіси, які виконують усі основні операції:

BookingService — створення і управління бронюваннями, перевірка доступності місць.

PaymentService — ініціація та підтвердження оплат.

EventService — отримання інформації про фільми, розклад і зали.

Основна функція: обробка запитів, перевірка даних, зв’язок між UI та БД.
Залежить від шару Data Access.

### 3)Data Access

Цей шар відповідає за роботу з базою даних та зберігання інформації:

BookingRepository — керування даними про бронювання.

TicketRepository — керування інформацією про квитки.

Database — основне сховище даних (таблиці користувачів, сеансів, бронювань, платежів).

Основна функція: виконання запитів до бази даних, збереження й отримання даних.
Не залежить від інших шарів.

## Взаємодія між шарами:

1)Presentation → Business Logic: користувач створює запит (наприклад, бронювання), який передається через контролери до сервісів.

2)Business Logic → Data Access: сервіси виконують бізнес-операції, звертаючись до репозиторіїв для зчитування або запису даних у базу.

3)Data Access → Database: репозиторії зберігають або отримують дані через SQL-запити.
